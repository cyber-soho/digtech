IPv4 vs IPv6

IPv4 (Legacy)

192.168.1.1

32-bit addresses. ~4.3 billion total. Exhausted. Relies on NAT (Network Address Translation) to survive.

IPv6 (Modern)

2001:0db8:85a3::8a2e

128-bit addresses. Undecillion addresses (3.4 √ó 10^38). No need for NAT. Built-in security features.

Web Mechanisms: Cookies

Understanding state on the stateless web. How sites remember you, track you, and the regulations (GDPR) protecting you.

Cookie Classifications

Session vs. Persistent ‚ñº

Session: Temporary. Stored in memory. Deleted when browser closes. Used for shopping carts, login checks.

Persistent: Stored on disk. Has an expiration date. Used for "Remember Me" and tracking across visits.

First-Party vs. Third-Party ‚ñº

First-Party: Set by the domain you are visiting (e.g., example.com). Essential for functionality.

Third-Party: Set by a different domain (e.g., ad-network.com) via an image or script. The engine of cross-site tracking and targeted ads.

Security Attributes ‚ñº
  • HttpOnly: Cannot be accessed by JavaScript (prevents XSS).
  • Secure: Only sent over HTTPS.
  • SameSite: Controls when cookies are sent with cross-site requests (prevents CSRF).

Service Models

IaaS (Infrastructure)

Raw compute (EC2, Azure VM). You manage OS, apps, data. Provider gives hardware.

PaaS (Platform)

Dev environment (Heroku, GAE). You manage code & data. Provider handles OS & runtime.

SaaS (Software)

Finished app (Gmail, Salesforce). You just use it. Provider manages everything.
